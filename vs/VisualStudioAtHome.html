Installing Visual Studio on your own machine

Note, this is not 100% required, all College (Windows) lab machines have Visual Studio 2012 installed and you may use these machines exclusively if you wish.  These instructions are for installing the current (free) version of Visual Studio 2013 on your home machine/laptop.

This document also includes some help on creating and running your programs.

Please note, this program only works on Windows 7 or Windows 8.  Mac OS X is not supported.

Step 1:  Download Visual Studio Express 2013 Desktop

http://www.visualstudio.com/en-us/products/visual-studio-express-vs.aspx

THIS IS EXTREMELY IMPORTANT.  You need to scroll past the Web and the Windows versions, you need to download Express 2013 for Windows Desktop

After you’ve selected Windows Desktop, click Install Now (do not choose the other options!)

At this time you may be required to sign in or create a Microsoft account.  The account is free.

Once you’ve signed in, you must again confirm that you are downloading the Express 2014 Windows Desktop edition.  Microsoft will try to get you to try the Ultimate Edition…. don’t.

Save the file (if your browser prompts for a location, use Desktop or somewhere else that you can find it easily).

Step 2:  Install Visual Studio

If you do not have Internet Explorer 10+, you will get this warning.  You can ignore it, and click continue.

Agree to the license and select “Install”

Unfortunately this next step is exceptionally slow…. just wait…. a while...

Eventually you will be asked to restart.  Do so.

Step 3:  Create a project

Launch Visual Studio

More waiting!

Once it opens, create a new C++ Empty Project

Please pay very careful attention to what you are doing when creating the project.  Failure to do ANY of the following will result in many issues!

Click OK - ignore the warning if you get it.

Step 4: Create a C++ source code file

Right click on Source Files, select Add, and then New Item.

Note - if your Solution Explorer is on the left side, that is fine.  If its not visible at all, select View/Solution Explorer from the top menu bar.

Select the correct file type (always C++ file (.cpp)).  You may name your file anything you wish, but it should be somewhat descriptive!

Enter in a basic C++ program

Step 5:  Build your code

Right click on your project name in the solution explorer.  Select build.

Note - if you have compiler errors, you will not be able to run your program.  You MUST correct them!

Step 6:  Run your code!

You’ll need to access the command prompt through your start menu.

As described in class, the command prompt is just a textual way to move through directories and run program.  You need to first make sure you are on the same drive as the program you just created.  For example, if you saved your program to your U: drive, you might need to enter U: <enter> to switch over to U drive from C drive.

Step 7:  Submitting your code

On homework assignments and during exams, you will be required to upload source code.  This is very important - Visual Studio creates many files and it is easy to upload the wrong one (which will get you precisely ZERO points!).

You must ALWAYS turn in the .cpp file (Source.cpp - if that is what you named it).  I will never ask you to upload any other files (including the .exe file).
